RETRO-GRPO (2026 - Ongoing)

Reinforcement learning for LLMs wastes a lot of exploration.

On difficult problems the model often keeps trying similar strategies and failing, with most of the learning happening indirectly through gradient updates.

There's already work on conditioning RL with prior experience - ICRL, SDFT, and iGRPO guide exploration using demonstrations or strong rollouts, and my additional views are probably not unique. These methods work well in many settings, but they can also sharpen the policy around specific solutions, narrowing exploration in a way that departs from the more open-ended, experiential learning style Sutton originally envisioned for RL.

I'm exploring a slightly different angle: conditioning on summaries of failed attempts.

After each round of attempts, a small summariser describes what strategies were tried and how they ended up. The model reads this before the next round of rollouts.

Instead of showing what worked, the context simply records what has already been tried. The model decides where to explore next.

Over time the summaries are removed so the behaviour transfers into the weights.

Parity (2026)

On every agent I've worked on, evals always lagged behind the agent's development. So I'm building a GitHub Action that watches changes to agent harnesses and suggests targeted evals.

The idea: if you change behaviour, you should automatically test what that new behaviour actually is.

It parses PR diffs, infers behavioural changes, retrieves similar evals (LangSmith etc.), and generates new probes—with human-in-the-loop approval.

Note: This is the last of my non "researchy" projects for the year.

Context Compression API (FoundersHack Hackathon Winner, 2026)

Built in ~48 hours.

Trained models to drop useless tokens from both:

  • video (frames / scenes)
  • text (filler tokens)

Goal: reduce cost without hurting downstream model performance.

Tried two approaches:

  • training-free (CLIP + scene segmentation)
  • RL-based compression

Also distilled a BERT-based token filter from LLM-Lingua-2 for lower latency.

Main constraint: compression must be cheaper than doing nothing.

AgentUQ (2026)

A simple framework for adding uncertainty quantification into agent workflows.

Abstracts:

  • algorithms
  • configuration
  • decision logic

Notes: Uncertainty quantification has always felt too "experimental" for adoption into actual agent stacks. But I figured they're lighter than judges and carry more semantics than formal verifiers, so they were at least worth a shot. The idea was to assess how sure an agent is before executing what it does, or listening to what it says. Basically I wanted to see what a system using UQ could look like, so I built a simple framework for adding uncertainty quantification into agent workflows with abstractions that might make it easier for developers to accept. In the end, the math underneath a lot of UQ methods, especially beneath LLMs tends to fall apart in practice. "Uncertainty" itself is of course a flawed metric to track in LLMs, as LLMs can hallucinate with confidence.
